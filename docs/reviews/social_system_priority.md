# 社交系统优先级复评

来源：`docs/archive/meow_pilot_architecture_todo.md`

## 结论

当前阶段不继续扩展真实 P2P 社交协议，优先保留本地账号、好友列表和本地交换记录。

## 原因

- 架构 review 将社交系统归入 P4，核心 P1/P2 能力仍是本地任务、日历、Surface、记忆和工具可靠性。
- 真实好友消息、技能交换和 P2P 网络协议需要身份、加密、同步、冲突处理和滥用防护，不适合作为当前主线的轻量修复。
- 原前端存在模拟好友消息和硬编码 peer skill 展示，容易让用户误以为已经接入真实网络能力。

## 已调整

- Friends 页面不再声称好友消息已发送；未接入运行时时会明确提示未发送。
- Skill 中心不再混入硬编码的额外 Codex skill 目录。
- 好友交换按钮只写入真实的本地 `social.exchange` 记录，不再展示伪造 peer skill 货架。

## 后续条件

只有在明确进入社交功能阶段时，再补齐：

- 好友消息 RPC 和持久化表。
- 技能交换请求、接受、撤销和审计记录。
- P2P 或中继协议设计。
- 端到端加密、权限边界和滥用防护。
